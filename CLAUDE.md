## Critical Rules (Always Apply)

- **§0.1 IMPORTANT:** This repo IS the Moberg.Warp framework. `IJob`/`IMediator`/`IRequest`/`IMessage`/`IStreamRequest` are DEFINED here (`Warp.Core/Handlers/`) — editing them changes the public contract. NEVER pull in `MediatR` as the framework mediator (it exists only in benchmarks).
- **§0.2 YOU MUST** keep the `FOR UPDATE SKIP LOCKED` raw SQL in `IWarpSqlQueries` implementations. It is intentional for atomic job dequeue — DO NOT rewrite it as LINQ or flag it as an anti-pattern.
- **§0.3** WHEN adding DB-dialect code, DO NOT put `UseNpgsql`/`UseSqlServer` or provider SQL in `Warp.Core` — it goes in `src/core/providers/Warp.Provider.*`, and must work on BOTH providers.
- **§0.4** Mediator/HTTP dispatch is compile-time source-generated (`netstandard2.0` generator projects). Rely on the generator; DO NOT add runtime reflection registration.
- **§0.5** WHEN a test needs relational behavior (SKIP LOCKED, transactions, snake_case), use the Testcontainers fixtures — NEVER `UseInMemoryDatabase`.
