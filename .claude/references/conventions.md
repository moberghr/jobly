## Naming Conventions

- **Namespaces:** file-scoped, always. Evidence: 610/610 `.cs` files with a namespace use `namespace X;` form; 0 block-scoped.
- **DI registration:** grouped in `*ServiceConfiguration.cs` or `Warp*Extensions.cs` — `Add*`/builder-extension style, not inline in host startup.
- **Source generators:** `Warp*Generator.cs` in the `*.SourceGenerator` (netstandard2.0) projects.
- **Tests:** `*Tests.cs` classes (e.g. `NamingConventionTests`), xUnit v3 `[Fact]`/`[Theory]`.

## File Organization

- Layer/responsibility-per-project under `src/core` (`Warp.Core`, `Warp.Worker`, `Warp.Http`, `Warp.UI`, `providers/*`); within a project, group by feature folder (`Handlers/`, `Data/Queries/`, `BackgroundServices/`, `DashboardPush/`).
- Provider-specific code (SQL dialects, DB config) lives ONLY under `src/core/providers/Warp.Provider.*` — keep `Warp.Core` provider-agnostic.

## DI Registration

- Use the existing `*ServiceConfiguration` / `WarpBuilder` extension pattern; do not register Warp services inline in consumer `Program.cs` beyond the public `AddWarp*` entry points.

## Response & Error Handling

- No repo-wide `Result<T>` envelope — code mostly throws or returns domain types. Don't introduce a Result envelope unless matching a local file that already uses one.

## Test Conventions

- xUnit v3 on Microsoft Testing Platform; assertions with **Shouldly** (`.ShouldBe(...)`), mocks with **Moq**, DB reset with **Respawn**.
- Relational tests run against real engines via **Testcontainers** (Postgres + SQL Server) with shared-container class fixtures (`src/tests/Warp.Tests/Fixtures/`). Do NOT default to `UseInMemoryDatabase` when relational behavior matters — SKIP LOCKED, transactions, and snake_case mapping require a real engine.

## Other Patterns

- Mediator/handler dispatch is compile-time source-generated — adding a handler means the generator wires it; avoid runtime reflection-based registration.
- DB naming is snake_case via `EFCore.NamingConventions`.
- Recurring schedules use `Cronos` cron expressions.
