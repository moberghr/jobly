## 3. Design Patterns in Use

### 3.1 Source-generated mediator
- [EXTRACTED] The framework defines its own `IMediator`, `IRequest`, `IRequestHandler`, `IJob`, `IMessage`, `IStreamRequest` and emits dispatch via Roslyn source generators (no `MediatR` at runtime). Evidence: `src/core/Warp.Core/Handlers/*.cs`; `src/core/Warp.SourceGenerator/WarpMediatorGenerator.cs`, `WarpMetadataGenerator.cs`.
- [EXTRACTED] `MediatR` and `Mediator.SourceGenerator` appear ONLY in the benchmark project for comparison — they are not framework dependencies. Evidence: `grep -rl MediatR --include=*.csproj` → only `src/benchmarks/Warp.Benchmarks/Warp.Benchmarks.csproj`.

## 5. Data Layer

- [EXTRACTED] EF Core is the job/message store; providers configure Npgsql or SQL Server. Evidence: `UseNpgsql`/`UseSqlServer` across `Warp.Core` and provider `*ServiceConfiguration.cs`.
