# Performance (§6)

> Generated by setup-bootstrap 2026-05-29. Cite as §6.N in reviews.
> See `.claude/references/dotnet/performance-supplement.md`.

- **§6.1** [CONVENTION] This is a throughput-sensitive job framework — the dequeue/dispatch hot path (`src/core/Warp.Worker/WarpDispatcher.cs`) must stay allocation-light and avoid per-job reflection. There are dedicated benchmark projects (`src/benchmarks/*`); run them when changing hot paths.
- **§6.2** [CONVENTION] Use async EF query methods (`ToListAsync`, `FirstOrDefaultAsync`) and propagate `CancellationToken` through the worker loop and handlers.
- **§6.3** [CONVENTION] Prefer the signal-driven wake-up path over polling where it already exists; don't reintroduce tight polling loops.
- **§6.4** [CONVENTION] Reuse pooled DbContext / shared connections via the existing service configuration — don't instantiate ad-hoc contexts in the dispatch loop.
