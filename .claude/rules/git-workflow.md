# Git & Workflow (§8)

> Generated by setup-bootstrap 2026-05-29. Cite as §8.N in reviews.

- **§8.1** [CONVENTION] Conventional commit prefixes are used: `feat:`, `fix:`, `docs:`, `test:`, `chore:`, often scoped (`fix(ui):`, `feat(http-sourcegen):`). Evidence: `git log --oneline`.
- **§8.2** [CONVENTION] Hierarchical branch names with `/`: `feat/...`, `fix/...`, `bug/...`, `chore/...`. Default branch is `main`; work merges via PR (commits carry `(#NNN)`).
- **§8.3** [ENFORCED] `TreatWarningsAsErrors=true` repo-wide — a build with any analyzer/compiler warning fails. Fix warnings, don't suppress globally; scoped suppressions go in the relevant `.editorconfig`.
- **§8.4** [CONVENTION] Releases ship via `release.yml`; release notes live under `website`/`docs`. Bump and document when releasing.
