# Testing (§4)

> Generated by setup-bootstrap 2026-05-29. Cite as §4.N in reviews.
> See `.claude/references/dotnet/testing-supplement.md`.

- **§4.1** [CONVENTION] Test framework is xUnit v3 (`xunit.v3.mtp-v2`) on the Microsoft Testing Platform runner (`global.json`). New tests use `[Fact]`/`[Theory]`, assert with **Shouldly** (`.ShouldBe(...)`), mock with **Moq**.
- **§4.2** [ENFORCED] WHEN a test exercises relational behavior (SKIP LOCKED, transactions, snake_case mapping, concurrency), use the **Testcontainers** Postgres/SQL Server fixtures (`src/tests/Warp.Tests/Fixtures/`, class fixtures) — DO NOT use `UseInMemoryDatabase`.
- **§4.3** [CONVENTION] DB state between tests is reset with **Respawn**; reuse the shared-container fixtures rather than spinning new containers per test.
- **§4.4** [CONVENTION] Match existing fixture/naming patterns in `src/tests/Warp.Tests/`; assertions should be meaningful, not just "does not throw".
