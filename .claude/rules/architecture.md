# Architecture (§2)

> Generated by setup-bootstrap 2026-05-29. Cite as §2.N in reviews.
> Full descriptive audit: `.claude/references/architecture-principles.md`.

- **§2.1** [ENFORCED] `Warp.Core` is provider-agnostic. WHEN you need PostgreSQL- or SQL-Server-specific code, put it in `src/core/providers/Warp.Provider.*` behind `IWarpSqlQueries` / a provider interface — DO NOT add `UseNpgsql`/`UseSqlServer` or dialect SQL to `Warp.Core`.
- **§2.2** [ENFORCED] Mediator/handler dispatch is compile-time source-generated (`Warp.SourceGenerator`, `Warp.Http.SourceGenerator`). WHEN adding a handler/endpoint, rely on the generator — DO NOT add runtime reflection-based registration or pull in `MediatR` (it exists only in benchmarks for comparison).
- **§2.3** [CONVENTION] Source-generator projects target `netstandard2.0`; everything else targets `net10.0`. Don't change a generator project's TFM to net10.0 — Roslyn analyzers/generators must load on netstandard2.0.
- **§2.4** [CONVENTION] Register services via `*ServiceConfiguration.cs` / `Warp*Extensions.cs` (the `AddWarp*` builder pattern), not inline in host startup.
- **§2.5** [CONVENTION] HTTP surface is minimal-API endpoints + `[WarpHttp]` source-gen, not MVC controllers. Match the existing endpoint style.
