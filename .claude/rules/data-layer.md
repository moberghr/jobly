# Data Layer (§5)

> Generated by setup-bootstrap 2026-05-29. Cite as §5.N in reviews.
> See `.claude/references/dotnet/ef-core-checklist.md`.

- **§5.1** [ENFORCED] The `FOR UPDATE SKIP LOCKED` / `FOR NO KEY UPDATE SKIP LOCKED` raw SQL in `IWarpSqlQueries` implementations is intentional and load-bearing for atomic job dequeue. DO NOT "refactor" it into LINQ or flag it as an anti-pattern. Evidence: `Warp.Core/Data/Queries/IWarpSqlQueries.cs:15-51`.
- **§5.2** [ENFORCED] DB naming is snake_case via `EFCore.NamingConventions` (`UseSnakeCaseNamingConvention`). New entities/columns inherit this — do not hand-name to PascalCase.
- **§5.3** [CONVENTION] Add `AsNoTracking()` for read-only EF queries; prefer `.Select()` projection over `.Include()` for reads.
- **§5.4** [CONVENTION] Provider-specific SQL stays in `Warp.Provider.PostgreSql` / `Warp.Provider.SqlServer`; `Warp.Core` consumes it through `IWarpSqlQueries`.
- **§5.5** [CONVENTION] Distributed coordination uses `DistributedLock.Postgres` / `DistributedLock.SqlServer` — reuse the existing lease coordinator rather than rolling new locking.
