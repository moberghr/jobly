# Project-Specific (§9)

> Generated by setup-bootstrap 2026-05-29. Cite as §9.N in reviews.

- **§9.1** [CONVENTION] **This repo IS the Moberg.Warp framework**, not a consumer of it. `IJob`, `IMediator`, `IRequest`, `IMessage`, `IStreamRequest` are defined HERE (`Warp.Core/Handlers/`). When changing them you are changing the public contract — treat as breaking-change-sensitive.
- **§9.2** [ENFORCED] Two `netstandard2.0` source-generator projects (`Warp.SourceGenerator`, `Warp.Http.SourceGenerator`) emit the mediator/HTTP dispatch. Generator changes need matching tests in `Warp.Tests.SourceGenerator`.
- **§9.3** [CONVENTION] The dashboard is a React 19 + Vite SPA under `src/ui` (axios/REST + `@microsoft/signalr`, TanStack Query/Table, Tailwind, shadcn, Zustand). UI work uses npm + Vite there; it is separate from the .NET build.
- **§9.4** [CONVENTION] Two DB providers (PostgreSQL, SQL Server) are first-class — a feature touching the data layer must work on both. Add provider SQL in BOTH `Warp.Provider.*` projects.
- **§9.5** [CONVENTION] `MediatR`/`Mediator.SourceGenerator` exist only in `Warp.Benchmarks` for comparison — never treat them as the framework's mediator.
