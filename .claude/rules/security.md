# Security (§1)

> Generated by setup-bootstrap 2026-05-29. Cite as §1.N in reviews.

- **§1.1** [ENFORCED] NEVER commit secrets, connection strings with credentials, or API keys. Demo connection strings stay in `appsettings*.json` / env, never hardcoded in `.cs`.
- **§1.2** [CONVENTION] Provider raw SQL (`IWarpSqlQueries` implementations) MUST stay parameterized. The `FOR UPDATE SKIP LOCKED` queries are intentional, but any new dynamic SQL must use parameters — never string-concatenate table/column names from user input.
- **§1.3** [CONVENTION] Dashboard HTTP/SignalR endpoints (`Warp.UI`, `Warp.Http`) are operational surfaces. WHEN adding a new endpoint, DO NOT expose job payloads or internal state without considering whether the consumer should authorize it.
- **§1.4** No PII in logs — job payloads can carry sensitive data; log identifiers, not full payloads.

See `.claude/references/security-checklist.md` for the full shared checklist.
